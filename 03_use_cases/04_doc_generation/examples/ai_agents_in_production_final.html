Production pattern

Code: Production Guardrails in LangGraph

agent_guardrails.py — Python 3.12
from langgraph.graph import StateGraph
from pydantic import BaseModel

class AgentState(BaseModel):
    iterations: int = 0
    max_steps: int = 8

def tool_node(state: AgentState):
    # 1. Strict schema validation + allowlist
    assert tool_call.name in ALLOWED_TOOLS
    result = execute_with_timeout(tool_call, 5.0)
    
    # 2. Explicit error handling
    if not result.ok:
        return {"error": result.error, "fallback": True}
    return result

# 3. Loop protection + cost control
graph.add_conditional_edges(
    "agent",
    lambda s: "end" if s.iterations >= s.max_steps else "tools"
)

# 4. Full tracing
graph = graph.compile(checkpointer=PostgresSaver())
🔒 Strict tool schemas and explicit allowlists prevent hallucinated calls
🔁 Iteration limits and timeouts stop infinite loops
💾 Full state checkpointing enables time-travel debugging

Reliability

Why Agents Fail in Production — Top 5 Failure Modes

🧨
Tool Misuse / Hallucinated Calls: Wrong tool or bad parameters — Fix: strict schemas, allowlists, validation
♾️
Infinite Loops / Over-Reasoning: Agents never converge — Fix: iteration limits, step caps, early-exit rules
🔍
Bad Retrieval Context: Irrelevant RAG → wrong answers — Fix: relevance scoring, task-specific context
🤫
Silent Failures: Tools fail partially, system proceeds anyway — RAG failures are "silent, hidden behind plausible text"
💸
Cost Spikes / Context Accumulation: Unbounded history hits 200k token limits — Fix: token budgets, summarization, sliding windows

Playbook

Production Best Practices — Building for Reliability

🔐
Harden tool use: Strict Pydantic schemas, allow-listed tools only, validate before execution, explicit error handling with fallbacks
🎛️
Control autonomy: Iteration limits, token budgets per-workflow, periodic summarization, early-exit rules
🧪
Test like software: Version prompts, trace every agent run, run scenario-based evaluations, treat RAG/tools as testable subsystems
👤
Keep humans in control: Targeted checkpoints, escalations, manual override, reversible actions, full audit logging
🌐
Use a multi-provider AI gateway with failover and cost metrics. Ship agents like you ship microservices.
